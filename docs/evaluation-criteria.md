# Критерии оценки — СМП

## Общие критерии (для всех ролей)

### 1. Функциональность сервиса (40 баллов)

| Критерий | Баллы | Описание |
|----------|:---:|---|
| Health-check работает | 5 | Сервис запускается в Docker и отвечает на `/health` |
| API реализованы | 15 | Все `[REQUIRED]` эндпоинты работают согласно контракту |
| Интеграция со смежными сервисами | 10 | Сервис успешно взаимодействует с соседями по цепочке |
| Обработка ошибок | 5 | Корректные HTTP-статусы и сообщения об ошибках |
| Валидация входных данных | 5 | Проверка обязательных полей, форматов, границ значений |

### 2. Качество кода (25 баллов)

| Критерий | Баллы | Описание |
|----------|:---:|---|
| Читаемость и стиль | 5 | Понятные имена, форматирование, комментарии |
| Архитектура | 10 | Разделение на слои, инверсия зависимостей, SOLID |
| Конфигурация | 5 | Параметры вынесены в конфиг (переменные окружения / config-файл) |
| Логирование | 5 | Осмысленные логи (INFO/WARN/ERROR), request-id tracing |

### 3. Тестирование и документация (20 баллов)

| Критерий | Баллы | Описание |
|----------|:---:|---|
| README.md сервиса | 5 | Назначение, endpoints, как запустить, конфигурация |
| Тесты (unit/integration) | 10 | Хотя бы 3 теста, покрывающих основной сценарий и граничные случаи |
| Dockerfile | 5 | Сервис собирается и запускается в контейнере |

### 4. Командная работа (15 баллов)

| Критерий | Баллы | Описание |
|----------|:---:|---|
| Соблюдение API-контракта | 5 | Сервис не отклоняется от утверждённой спецификации |
| Git-дисциплина | 5 | Осмысленные коммиты, ветки, pull requests |
| Взаимодействие с командой | 5 | Участие в синхронизациях, помощь смежным сервисам |

### Итого: 100 баллов

| Диапазон | Оценка |
|----------|:---:|
| 90–100 | Отлично |
| 75–89 | Хорошо |
| 60–74 | Удовлетворительно |
