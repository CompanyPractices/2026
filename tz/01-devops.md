# Техническое задание: DevOps

**Роль:** DevOps-инженер — инфраструктура, CI/CD и API-контракты
**Количество человек:** 1
**Сложность:** ⭐⭐⭐
**Примерный объём кода:** 300–700 строк (YAML, Dockerfile, CI/CD, Shell)

---

## Твоя задача

Ты отвечаешь за инфраструктуру всего проекта `{PROJECT_NAME}`: Docker Compose, CI/CD и централизованную OpenAPI-спецификацию. Все 8 микросервисов должны запускаться одной командой и работать как единая система.

> **Организационная часть** (координация команд, ежедневные синхронизации, отслеживание прогресса, фиксация названия проекта) — выполняется куратором практики. Твоя роль — чисто техническая.

---

## Обязательные требования

### 1. Docker Compose (главный артефакт)

Файл `docker-compose.yml` в корне репозитория должен:

- Поднимать все 8 микросервисов (плюс 1 инфраструктурный PostgreSQL, итого 9 контейнеров):
  - Gateway (8080)
  - Card Management (8081)
  - Switch (8082)
  - Authorization (8083)
  - Terminal Simulator (8085)
  - Merchant + Acquirer Simulator (8086)
  - Transaction Logger (8088)
  - Web Dashboard (3000)
- Поднимать инфраструктурный сервис:
  - PostgreSQL (5432)
- Настроить сеть (одна общая сеть)
- Настроить healthcheck (каждый сервис проверяется через `/health`)
- Настроить зависимости (сервисы ждут доступности БД перед стартом)
- Все переменные окружения (порты, хосты БД, ключи) вынесены в `.env` файл

**Команда запуска всей системы:**

```bash
docker compose up -d
```

### 2. API-контракты

- Собрать и зафиксировать API-контракты со всех команд в единую OpenAPI 3.0 спецификацию
- Утвердить контракты до начала кодирования (день 2)
- Следить за соблюдением контрактов (несоответствие = баг)

### 3. CI/CD (GitHub Actions)

Файл `.github/workflows/ci.yml`:

- Триггер: push в `main`, pull request
- Шаги:
  - Проверка стиля кода (линтер для каждого языка)
  - Сборка всех сервисов
  - Прогон unit-тестов
  - Сборка Docker-образов (опционально)

### 4. Репозиторий

- Структура согласно общему README
- Ветки: `main` (стабильная), feature-ветки для каждой команды
- Защита `main` ветки (PR + review)
- `.gitignore` настроен

### 5. Помощь командам с Dockerfile

- Каждый сервис должен иметь свой `Dockerfile`
- Помогай командам с написанием и отладкой Dockerfile
- Проверяй, что все контейнеры корректно работают в общей сети Docker Compose

## Дополнительные задания (бонусные баллы)

- Prometheus + Grafana для мониторинга всех сервисов
- ELK / Loki для централизованного сбора логов
- Автоматическое масштабирование (scale для симуляторов)
- Makefile с полезными командами (`make run`, `make test`, `make clean`)
