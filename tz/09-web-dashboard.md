# Техническое задание: Web Dashboard

**Роль:** Frontend — веб-интерфейс для визуализации работы процессинга
**Количество человек:** 2
**Сложность:** ⭐⭐⭐
**Примерный объём кода:** 600–1200 строк (TypeScript/React)

---

## Ваша задача

Web Dashboard — это лицо `{PROJECT_NAME}`. Вы создаёте SPA (Single Page Application), которое в реальном времени показывает поток транзакций, статистику и графики. Это то, что увидят все на финальной презентации.

Ваш дашборд должен быть наглядным, информативным и впечатляющим.

---

## Обязательные требования

### 1. Технологический стек

- **React 18** (или Vue 3 / Angular — но React рекомендуется)
- **TypeScript**
- CSS Framework на выбор: Tailwind CSS, Ant Design, MUI
- Библиотека графиков: Chart.js (react-chartjs-2) или Recharts
- WebSocket-клиент (нативный WebSocket API)

### 2. Основные виджеты дашборда

Дашборд состоит из следующих виджетов:

#### A. Хедер-статистика (4 карточки KPI)

```text
┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│   1,250      │ │     88%      │ │  1,875,000 ₽ │ │    38 ms     │
│  Всего TX    │ │  Approval    │ │  Общая сумма  │ │  Среднее     │
│              │ │    Rate      │ │               │ │  время       │
└──────────────┘ └──────────────┘ └──────────────┘ └──────────────┘
```

Данные: `GET /api/dashboard/stats`

#### B. График потока транзакций (линейный)

Показывает количество транзакций в минуту за последний час. Обновляется в реальном времени.

```text
  TX/min
    ↑
 15 ┤        ╭──╮
 10 ┤   ╭────╯  ╰───╮
  5 ┤───╯            ╰───
  0 ┼───────────────────────────→ время
   09:00  09:15  09:30  09:45
```

Данные: WebSocket (`/ws/transactions`) — агрегируете на клиенте.

#### C. Круговая диаграмма Approved vs Declined

```text
    ┌──────────┐
   ╱    ██     ╲
  │    ████  88%│  ██ Approved (1100)
  │    ████     │  ░░ Declined (150)
   ╲    ░░     ╱
    └──────────┘
```

#### D. Таблица последних транзакций

Таблица с последними 20 транзакциями. Обновляется в реальном времени через WebSocket.

| Время | PAN | Сумма | Мерчант | Статус |
|-------|-----|:---:|---------|:---:|
| 14:30:01 | 4000****0001 | 1 250 ₽ | Пятёрочка #1234 | ✅ |
| 14:30:02 | 4001****0002 | 450 ₽ | Вкусно и точка | ✅ |
| 14:30:03 | 4000****0003 | 85 000 ₽ | М.Видео | ✅ |
| 14:30:04 | 4000****0001 | 75 000 ₽ | Ozon.ru | ❌ |

- PAN маскируется: показываем первые 4 и последние 4 цифры (`4000****0001`)
- Статус: зелёный индикатор для APPROVED, красный для DECLINED
- При клике на строку — модальное окно с деталями транзакции

> **Конвертация сумм:** все API возвращают суммы в копейках (minor units). Для отображения в рублях делите на 100: `amount / 100`. Пример: `150000` → `1 500.00 ₽`.

#### E. Фильтры

Над таблицей — фильтры:
- По статусу: Все / Approved / Declined
- По дате: date picker (с-до)
- По BIN/эмитенту: выпадающий список
- По MCC: выпадающий список

### 3. Real-time через WebSocket

Подключение к `ws://localhost:8088/ws/transactions` (напрямую к Transaction Logger, не через Gateway — WebSocket-соединения не проксируются через Gateway для простоты). Каждое новое сообщение:
- Добавляется в начало таблицы
- Обновляет KPI-карточки
- Обновляет линейный график
- Обновляет круговую диаграмму

### 4. Детали транзакции (модальное окно)

При клике на строку таблицы — модальное окно:

```text
┌─────────────────────────────────────┐
│  Transaction Details                │
│                                     │
│  STAN:       000001                 │
│  RRN:        012345678901           │
│  PAN:        400000****0001         │
│  Amount:     1 250.00 ₽            │
│  Status:     ✅ APPROVED            │
│  Auth Code:  ABC123                 │
│                                     │
│  Terminal:   TERM001 (POS)          │
│  Merchant:   Пятёрочка #1234       │
│  MCC:        5411 (Супермаркеты)    │
│  Acquirer:   ACQ001                 │
│  Issuer:     ISS001                 │
│                                     │
│  Time:       14:30:01               │
│                                     │
│                         [× Закрыть] │
└─────────────────────────────────────┘
```

### 5. Автообновление и реконнект

- При обрыве WebSocket — автоматический реконнект с exponential backoff
- Индикатор состояния соединения (зелёный/красный кружок в углу)
