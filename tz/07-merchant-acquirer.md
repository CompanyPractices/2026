# Техническое задание: Merchant + Acquirer Simulator

**Роль:** Эмулятор мерчантов и эквайреров
**Количество человек:** 1
**Сложность:** ⭐⭐⭐
**Примерный объём кода:** 300–500 строк (Python/Java/Go)

---

## Твоя задача

Merchant + Acquirer Simulator эмулирует поведение торговых точек и банков-эквайреров. В отличие от Terminal Simulator (который фокусируется на картах и типах транзакций), ты добавляешь разнообразие по типам бизнеса (MCC-коды), категориям и комиссиям эквайрера.

Ты делаешь систему реалистичнее: в реальном мире транзакции идут из супермаркетов, ресторанов, интернет-магазинов, и каждый тип бизнеса имеет свою комиссию.

---

## Обязательные требования

### 1. Health-check: `GET /health`

```json
{
  "status": "ok",
  "service": "merchant-acquirer-simulator",
  "merchantsLoaded": 20
}
```

### 2. Запуск симуляции: `POST /api/simulator/merchant/run`

**Вход:**

```json
{
  "count": 50,
  "scenario": "grocery"
}
```

Если указан `scenario`, используются MCC-коды, соответствующие этому сценарию. Для явного переопределения MCC можно использовать опциональное поле `mccCodes`:

```json
{
  "count": 50,
  "scenario": "grocery",
  "mccCodes": ["5411", "5499"]
}
```

> **Правило приоритета:** если указан `mccCodes` — используются явно переданные MCC-коды, `scenario` при этом определяет только временной диапазон и распределение сумм. Если `mccCodes` не указан — используются MCC по умолчанию для заданного `scenario`.

**Выход:** Аналогично Terminal Simulator — статистика approved/declined.

### 4. MCC-коды (Merchant Category Code)

Используемые MCC:

| MCC | Категория | Средний чек | Комиссия |
|:---:|---|:---:|:---:|
| 5411 | Супермаркеты | 500–1500 ₽ | 1.5% |
| 5812 | Рестораны / фастфуд | 300–1500 ₽ | 2.5% |
| 5814 | Fast Food | 200–500 ₽ | 2.0% |
| 5732 | Электроника | 1000–50 000 ₽ | 1.8% |
| 5399 | E-commerce (общее) | 500–10 000 ₽ | 2.0% |
| 4814 | Телеком-услуги | 300–1000 ₽ | 1.0% |
| 7994 | Развлечения / игры | 500–5000 ₽ | 3.0% |
| 3501 | Отели / гостиницы | 3000–15 000 ₽ | 3.5% |

### 5. Сценарии

#### `grocery` — продуктовые магазины
- MCC: 5411, 5499
- Суммы: 100–3000 ₽
- Время: 08:00–23:00
- 95% одобрений

#### `electronics` — электроника
- MCC: 5732, 5045
- Суммы: 5000–100 000 ₽
- Время: 10:00–22:00
- 90% одобрений (чаще срабатывают лимиты)

#### `restaurant` — рестораны
- MCC: 5812, 5814
- Суммы: 500–5000 ₽
- Время: 11:00–01:00

#### `travel` — путешествия
- MCC: 3501, 4511, 4722
- Суммы: 5000–50 000 ₽
- Разные времена
- Крупные суммы — проверка лимитов

### 6. Расчёт комиссии эквайрера

При генерации транзакции вычисляй комиссию эквайрера:

```python
def generate_transaction(merchant, card):
    amount = random.randint(
        int(merchant.average_check * 0.5),
        int(merchant.average_check * 2.0)
    )
    acquiring_fee = int(amount * merchant.acquiringFee)  # в копейках

    tx = {
        "mti": "0100",
        "stan": f"{stan_counter:06d}",
        "pan": card["pan"],
        "processingCode": "000000",
        "amount": amount,
        "currencyCode": "643",
        "transmissionDateTime": datetime.now().isoformat(),
        "terminalId": f"TERM{random.randint(1, 99):03d}",
        "merchantId": merchant["id"],
        "mcc": merchant["mcc"],
        "acquirerId": merchant["acquirerId"],
    }
    # acquiringFee добавляется после отправки в Gateway (из ответа)
    # или передаётся отдельным полем, если Gateway поддерживает расширенную модель Transaction
    return tx, acquiring_fee
```

Поле `acquiringFee` — справочное, вычисляется симулятором и сохраняется в логе транзакции (см. модель [`Transaction`](docs/api-spec.md)). Не передаётся в теле AuthorizationRequest, идущем в Gateway.

### 7. Отправка транзакций

Так же, как Terminal Simulator: `POST http://gateway:8080/api/transactions`.
